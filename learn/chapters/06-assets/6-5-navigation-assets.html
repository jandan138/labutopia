Level5 navigation 不只依赖 navigation_lab USD，还依赖 barrier image 和路径规划工具。这里的资产开始从纯 3D 场景扩展到 planning representation。 换成更直白的话：这一节是在告诉你“遇到 Navigation assets：barrier map 与移动任务 时，应该先看哪一层，不要一上来被术语吓住”。

为什么它在本项目里重要

config/navigation/navigation_assets.yaml 指向 assets/navigation/barrier/lab_1.png。Navigation task/controller 结合 A* 工具与 ridgebase controller，把机器人从一个位置移动到目标位置。barrier image 可以理解成给 planner 用的简化 occupancy 信息。

从 RL 看 把概念落到 trajectory

这里的关键不是把术语背下来，而是看它怎样改变 episode、observation、action、success 和 dataset distribution。

从代码和资产落地

这提醒我们，资产不一定都是 USD。对于 navigation，2D barrier map、起终点、base controller 参数同样是 benchmark asset 的一部分。如果只复制 USD 而忽略 barrier map，mobile task 可能根本无法规划。

读论文和跑实验时怎么用

与 EBench 对比时，mobile manipulation 的场景资产、base motion schema、evaluation worker 和 dataset format 都要一起看，不能只比较是否有移动底盘。

本节要记住
  • Navigation asset 包括 USD 和 planning map。
  • Barrier image 是 task contract 的一部分。
  • Mobile task 的复用风险高于 table-top object asset。
