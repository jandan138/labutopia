SimulationApp, World, Stage 三件事

LabUtopia 的第一行关键代码不是 robot，也不是 task，而是 SimulationApp。它决定 Isaac Sim runtime 能否启动，后面的 World 与 Stage 都在它之后才有意义。

先讲人话

LabUtopia 的第一行关键代码不是 robot，也不是 task，而是 SimulationApp。它决定 Isaac Sim runtime 能否启动，后面的 World 与 Stage 都在它之后才有意义。 换成更直白的话：这一节是在告诉你“遇到 SimulationApp, World, Stage 三件事 时，应该先看哪一层，不要一上来被术语吓住”。

你可以这样想

把它想成一场舞台剧：Stage 是舞台，World 是每一拍的节奏，SimulationApp 是剧场电源。

读完你应该能回答

  • 这节和一次 episode、一个 asset 或一条数据有什么关系？
  • 如果要在项目里找证据，第一眼应该打开 main.py。
  • 如果这里出错，后面的 task、controller、dataset 或 EBench 复用会受什么影响？

为什么它在本项目里重要

SimulationApp 是应用级入口，负责加载 Omniverse/Isaac Sim 扩展、渲染和物理 runtime。World 是 Isaac Sim core API 里的 simulation wrapper，管理 physics step、reset、play/stop 状态。Stage 则是 OpenUSD 场景图，里面有 prim、reference、material、camera、robot articulation。

从 RL 看 把概念落到 trajectory

这里的关键不是把术语背下来，而是看它怎样改变 episode、observation、action、success 和 dataset distribution。

从代码和资产落地

本地 main.py 的顺序正好体现这个关系：先创建 SimulationApp，再 import Isaac Sim 相关模块，之后创建 World，获取 stage，然后用 add_reference_to_stage 把配置里的 USD 加到 /World。这也是很多 Isaac Sim 项目要求“先启动 SimulationApp 再 import omni”的原因。

main.py
main.py
simulation_app = SimulationApp(simulation_config)
world = World(stage_units_in_meters=1.0, physics_prim_path="/physicsScene", backend="numpy")
stage = omni.usd.get_context().get_stage()
add_reference_to_stage(usd_path=os.path.abspath(cfg.usd_path), prim_path="/World")

读论文和跑实验时怎么用

从 GAMES103 看，World.step 就像推进一次物理系统；从 USD 看，Stage 是状态的可编辑数据库；从 policy 看，二者共同产生下一步 observation。

本节要记住
  • SimulationApp 是应用 runtime。
  • World 是 simulation stepping wrapper。
  • Stage 是 USD scene graph。
