Level1 atomic skill：pick/place/pour/stir 的意义

Level1 task 看似简单，却是理解所有高阶任务的基础。它们把一个动作技能拆到足够短，让你能定位 robot、object、camera、success condition 哪一层出错。

先讲人话

Level1 task 看似简单，却是理解所有高阶任务的基础。它们把一个动作技能拆到足够短，让你能定位 robot、object、camera、success condition 哪一层出错。 换成更直白的话：这一节是在告诉你“遇到 Level1 atomic skill：pick/place/pour/stir 的意义 时，应该先看哪一层，不要一上来被术语吓住”。

你可以这样想

把它想成学做实验动作：先练拿起、放下、倾倒，再把它们串成完整实验。

读完你应该能回答

  • 这节和一次 episode、一个 asset 或一条数据有什么关系？
  • 如果要在项目里找证据，第一眼应该打开 config/level1_pick.yaml。
  • 如果这里出错，后面的 task、controller、dataset 或 EBench 复用会受什么影响？

为什么它在本项目里重要

本地 Level1 包含 pick、place、open/close door/drawer、pour、press、shake、stir 等配置。它们通常使用 Franka 和 chemistry lab 场景，episode horizon 较短，controller 逻辑也更直接。对初学者来说，先跑 level1_pick 比直接跑 long-horizon task 更适合建立 debug 感。

从 RL 看 把概念落到 trajectory

这里的关键不是把术语背下来，而是看它怎样改变 episode、observation、action、success 和 dataset distribution。

从代码和资产落地

从 data perspective 看，atomic skill demonstration 是组合任务的素材。即使未来用 VLA 或 Diffusion Policy，也要知道模型学到的动作 chunk 其实来自这些低层数据分布。

读论文和跑实验时怎么用

如果你想验证环境，建议先从 Level1 开始：camera 是否有图、robot 是否能动、object path 是否正确、HDF5 是否写出。

本节要记住
  • Atomic task 是 debugging unit。
  • 短 horizon 降低 compounding error。
  • 它们为组合技能提供动作和视觉分布。
