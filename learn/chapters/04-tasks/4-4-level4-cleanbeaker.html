先讲人话

Long-horizon task 最适合暴露 embodied agents 的真实短板：不是不会抓，而是步骤多、误差积累、状态切换和成功条件都更脆弱。 换成更直白的话：这一节是在告诉你“遇到 Level4 long horizon：CleanBeaker 作为案例 时，应该先看哪一层，不要一上来被术语吓住”。

你可以这样想

把它想成学做实验动作：先练拿起、放下、倾倒，再把它们串成完整实验。

读完你应该能回答

  • 这节和一次 episode、一个 asset 或一条数据有什么关系？
  • 如果要在项目里找证据，第一眼应该打开 config/level4_CleanBeaker.yaml。
  • 如果这里出错，后面的 task、controller、dataset 或 EBench 复用会受什么影响？

为什么它在本项目里重要

level4_CleanBeaker.yaml 使用 assets/chemistry_lab/hard_task/Scene1_hard.usd，controller_type 是 cleanbeaker。它不是单一动作，而是科学实验语境中的连续过程：定位器材、拿取、移动、清洗或相关操作、判断完成。

从 RL 看 把概念落到 trajectory

这里的关键不是把术语背下来，而是看它怎样改变 episode、observation、action、success 和 dataset distribution。

从代码和资产落地

从 policy 角度，long horizon 会放大 covariate shift。ACT 的 action chunking 和 Diffusion Policy 的 receding horizon 都试图缓解这个问题，但前提是 demonstration 数据覆盖了合理的中间状态。

读论文和跑实验时怎么用

读 CleanBeaker controller 时，不要只看最终 success。要记录每个 phase 的进入条件、目标 prim、末端位姿和 failure reason，这些才是训练失败后能解释现象的线索。

本节要记住
  • Long horizon 的主要风险是 phase drift。
  • Hard task scene 与普通 lab scene 不同，asset path 也不同。
  • 失败诊断要按阶段记录。
