先讲人话

目录不是简单罗列文件，而是按“论文概念 → 仿真栈 → runtime → task/controller → data/policy → assets → EBench 对比”的依赖顺序展开。 换成更直白的话：这一节是在告诉你“遇到 本书目录如何组织 时，应该先看哪一层，不要一上来被术语吓住”。

你可以这样想

把它想成先画地图：你不是马上拧螺丝，而是先知道门、桌子、机器人、数据出口分别在哪里。

读完你应该能回答

  • 这节和一次 episode、一个 asset 或一条数据有什么关系？

从代码和资产落地
