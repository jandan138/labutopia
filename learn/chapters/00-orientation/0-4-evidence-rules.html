先讲人话

项目理解最怕把论文愿景、本地 README、外部 benchmark 文档和代码事实混成一句话。本书会把来源口径拆开。 换成更直白的话：这一节是在告诉你“遇到 资料口径：哪些是论文事实，哪些是本地事实 时，应该先看哪一层，不要一上来被术语吓住”。

你可以这样想

把它想成先画地图：你不是马上拧螺丝，而是先知道门、桌子、机器人、数据出口分别在哪里。

读完你应该能回答

  • 这节和一次 episode、一个 asset 或一条数据有什么关系？
  • 如果要在项目里找证据，第一眼应该打开 README.md。
  • 如果这里出错，后面的 task、controller、dataset 或 EBench 复用会受什么影响？

为什么它在本项目里重要

LabUtopia 的公开资料来自 arXiv、NeurIPS 2025 abstract、project page、Hugging Face dataset 和本地 README。它们告诉我们 LabUtopia 面向 scientific embodied agents，强调 high-fidelity simulation、hierarchical benchmark、scientific laboratory tasks，并在本地 README 中给出 Isaac Sim 5.1、Python 3.11、Ubuntu 24.04 的环境要求。

从 RL 看 把概念落到 trajectory

这里的关键不是把术语背下来，而是看它怎样改变 episode、observation、action、success 和 dataset distribution。

从代码和资产落地

EBench 的公开资料来自 GitHub、EBench docs、Hugging Face EBench-Assets 与 EBench-Dataset，以及 GenManip 论文。它们告诉我们 EBench 是基于 Isaac Sim 4.1 与 GenManip server 的 indoor VLA manipulation benchmark，核心本体是 lift2 dual-arm mobile base。

读论文和跑实验时怎么用

本地代码事实优先用于解释 LabUtopia repo 现在怎么跑：例如 task/controller factory、HDF5 数据保存、RMPFlow controller、navigation barrier image。外部资料优先用于解释 benchmark relation，而不是替本地代码编不存在的接口。

本节要记住
  • 遇到版本冲突时，按“本地 README/代码事实”和“外部 EBench 文档事实”分别陈述。
  • 资产复用结论必须同时检查 simulation version、embodiment、task/eval contract、dataset format。
  • 所有外部来源统一放在 Appendix 参考资料。
